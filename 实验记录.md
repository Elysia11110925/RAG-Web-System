## 三、系统架构

```
浏览器 (index.html)
  │  POST /api/upload        ← 上传文档 (multipart/form-data)
  │  GET  /api/documents      ← 列出已上传文档
  │  DELETE /api/documents/<id> ← 删除文档
  │  POST /api/chat           ← 提问 (SSE 流式响应)
  │
  ▼
Flask 后端 (app.py :5002)
  │
  ├─ 文档处理 Pipeline
  │   1. pdfplumber 解析 PDF / 直接读取 TXT
  │   2. LangChain RecursiveCharacterTextSplitter 切块 (chunk_size=500, overlap=50)
  │   3. sentence-transformers all-MiniLM-L6-v2 向量化 (384维)
  │   4. ChromaDB PersistentClient 存储 + 余弦相似度索引
  │
  └─ 问答 Pipeline
      1. 相同模型将问题向量化
      2. ChromaDB Top-K 检索 (k=4)
      3. 拼接 Prompt 模板 (System + Context + Question)
      4. DeepSeek API 流式生成 (OpenAI 兼容接口)
      5. SSE 推送至前端，逐字渲染打字机效果
```

## 四、技术栈

### 4.1 文档处理 Pipeline

| 阶段 | 技术选型 | 说明 |
|------|------|------|
| 文档解析 | pdfplumber (PDF) / Python内置 (TXT) | pdfplumber已安装，支持中英文PDF提取 |
| 文本切块 | LangChain RecursiveCharacterTextSplitter | chunk_size=500, overlap=50，支持中英文分隔符 |
| 向量化嵌入 | sentence-transformers all-MiniLM-L6-v2 | 384维向量，GPU推理，首次运行自动下载(~80MB) |
| 向量存储 | ChromaDB (PersistentClient) | 本地持久化至chroma_db/目录，重启不丢失 |

### 4.2 问答 Pipeline

| 阶段 | 技术选型 | 说明 |
|------|------|------|
| 问题向量化 | 同 Embedding 模型 | 与文档使用同一模型，确保向量空间一致 |
| 相似度检索 | ChromaDB 余弦相似度 Top-K (K=4) | 余弦相似度，返回最相关的4个文档片段 |
| Prompt拼接 | 自定义 Prompt 模板 | System角色设定 + 参考资料 + 用户问题 |
| 答案生成 | DeepSeek API (OpenAI 兼容) | temperature=0.3减少幻觉，max_tokens=2048，流式输出 |

## 五、核心实现

### 5.1 后端 (app.py, 401行)

| 项目 | 详情 |
|------|------|
| 框架 | Flask 3.1.3 + flask-cors |
| 端口 | 5002 |
| 文档上传 | `POST /api/upload` — 接收PDF/TXT文件，解析→切块→向量化→存入ChromaDB |
| 文档列表 | `GET /api/documents` — 返回已上传文档名、切块数、大小、上传时间 |
| 文档删除 | `DELETE /api/documents/<id>` — 删除文档文件及其在ChromaDB中的所有向量 |
| RAG问答 | `POST /api/chat` — 接收问题→检索→Prompt拼接→DeepSeek流式生成→SSE返回 |
| 健康检查 | `GET /api/health` — 返回设备类型、模型状态、文档数量、LLM配置 |
| 流式方式 | SSE (Server-Sent Events) |
| 上传限制 | 32MB |

### 5.2 Prompt 模板

```
System: 你是一个基于知识库的问答助手。请根据以下参考资料回答用户问题。如果参考资料中没有相关信息，请如实告知。回答时请引用来源。\n\n参考资料：\n{context}\n\n用户问题：{question}\n\n回答：
```

### 5.3 前端 (index.html, 787行)

- 暗色主题界面：与ABSA/VQA统一的暗橙黑配色 (#0f0f1a背景, #f97316橙色点缀)
- 拖拽上传：支持点击选择或拖拽文件，实时显示上传状态
- 对话气泡：用户消息右侧橙色气泡，助手消息左侧深灰气泡，显示参考来源
- 流式打字机效果：SSE逐token接收，前端逐字渲染，模拟打字机效果
- 参考来源标注：回答底部显示引用的文档名称和片段数
- 响应式布局：移动端自动切换为上下布局，适配小屏幕

## 八、问题与解决方案

| # | 问题 | 原因 | 解决方案 |
|---|------|------|------|
| 1 | ChromaDB 1.5.x API不兼容 | 自定义Embedding类缺少embed_query方法，且__call__参数名需为input | 改用ChromaDB内置的SentenceTransformerEmbeddingFunction |
| 2 | 中文文件名上传崩溃 (IndexError) | Werkzeug secure_filename将中文文件名过滤为空字符串，.rsplit('.', 1)[1]越界 | 先用os.path.splitext提取原始扩展名，再用UUID生成安全文件名保存，前端显示保留原始名 |
| 3 | 嵌入模型首次加载慢 | all-MiniLM-L6-v2约80MB，首次需从HuggingFace下载 | 应用启动时预加载，后续请求直接使用；设置HF镜像加速 |
| 4 | ChromaDB持久化目录残留导致启动失败 | 上次异常退出时embedding function配置与新的不匹配 | 启动前清理chroma_db目录，或使用get_or_create_collection自动处理 |
| 5 | DeepSeek API Key需要手动配置 | API Key需用户自行注册获取 | 通过环境变量DEEPSEEK_API_KEY传入，使用setx永久保存 |
